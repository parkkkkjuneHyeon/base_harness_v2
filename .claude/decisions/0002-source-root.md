# 결정: 제품 코드는 src/ 아래에서 관리
날짜: 2026-07-03
상태: 확정

## 배경

저장소 루트에 하네스 설정(`.claude/`)과 실제 제품 코드가 섞여 있으면, 시간이 지날수록 하네스가
새 메타 폴더(예: `runs/`)를 추가할 때마다 "이건 코드가 아니다"를 훅·서브에이전트 쪽에서 계속
예외 목록으로 걸러줘야 하는 문제가 있었다.

## 결정

실제 제품 코드는 `src/`(=`.claude/project.yaml`의 `project.source_root`, 기본값 `"src"`) 아래에
서만 둔다. 저장소 루트(`.claude/`, `CLAUDE.md`, `runs/`)는 하네스 자체다.

`source_root` 값은 오케스트레이터(`/extend`)만 알고 사용한다 — 새 파일 배치, 컨벤션 스캔 범위,
영향범위 검색 범위를 정할 때 이 값을 기준으로 삼는다. `code-generator`/`impact-analyzer`
서브에이전트 정의에는 이 값을 넣지 않는다 — 경로 결정 권한은 오케스트레이터 한 곳에만 둔다는
기존 아키텍처 원칙(정책 계층 06절)과 일치시키기 위해서다.

## 근거

- Python 표준 관행(src-layout)과 맞다 — 저장소 루트에서 우연히 import되는 걸 막는다.
- 제외 목록(exclude-list) 대신 허용 목록(source_root) 방식이 더 안전하다 — 하네스에 메타 폴더가
  늘어나도 매번 훅의 제외 목록을 갱신할 필요가 없다.
- 경로 결정을 오케스트레이터 한 곳에 집중시키면, 서브에이전트가 임의로 파일을 엉뚱한 곳에 만드는
  사고를 구조적으로 막을 수 있다.
